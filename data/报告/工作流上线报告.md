写了一个全自动工具：贴入微信文章链接 → 自动翻译成英文 → 自动提 PR 到 [deepmodeling/blog](https://github.com/deepmodeling/blog)。一键完成。

​**代码仓库**​: https://github.com/Ahri-qwq/translate

## **📝 处理流程**

```Plain
[Step 1/6] 下载文章
    ├── 提取正文、标题、公式
    ├── 智能图片重编号（按出现顺序 pic1/pic2/...）
    ├── 跳过音视频内容并标注
    └── 生成中文 Markdown + images_map.txt

[Step 2/6] LLM 翻译 + 审核
    ├── 中文 → 英文翻译（保留公式、图片、Markdown 语法）
    ├── LLM 审核（漏译 / 术语 / 格式）
    ├── 交叉审查（可选）：双模型独立审查，合并报告
    └── 未通过时自动修改译文（最多 N 轮，可配置）

[Step 3/6] 保存文件
    ├── {标题}_zh.md — 中文原文
    ├── {标题}_en.md — 英文译文（含翻译元数据头）
    ├── {标题}_review.md — 审核报告
    ├── {标题}.images_map.txt — 图片 URL 映射
    └── images/ — 下载的正文图片

[Step 4/6] OSS 上传（可选）
    ├── 上传 images/ 中所有图片到 OSS
    ├── 更新 images_map.txt 为 OSS HTTPS 地址
    └── 上传失败时自动回退到原始微信 CDN URL

[Step 5/6] 格式转换 → 博客格式
    ├── LLM 自动分类（categories）
    ├── 清理翻译元数据块（> Source, > Translated by 等）
    ├── LaTeX 公式 → codecogs SVG 图片
    ├── 图片语法转换（![](local) → <center><img src=url /></center>）
    ├── 图N → Figure N 自动翻译
    ├── 图片间加粗文本 → ## 二级标题
    ├── 自动插入 <!-- more -->（第一段落后）
    ├── 自动检测 mathjax: true
    └── 文件命名：{CATEGORY}_{YYYY}_{MM}_{DD}.md

[Step 6/6] 上传 GitHub → PR
    ├── 检查 gh CLI 可用 & 已登录
    ├── 同步本地缓存仓库（git pull origin master）
    ├── 创建分支 post-{CATEGORY}_{YYYY}_{MM}_{DD}
    ├── 复制 blog_output/*.md → source/_posts/
    ├── git commit + git push
    └── gh pr create → 返回 PR 链接
```

## **怎么跑**

```Bash
pip install -r requirements.txt
copy config.yaml.example config.yaml   # 填入 API Key
python src/main.py                     # 交互模式，贴链接即可
```

交互就行了。

## 项目亮点

- **一键到底**：贴链接 → 自动完成 6 步流程，从下载到 PR 全程无需手动操作
- **双模型交叉审查**：翻译完成后两个不同 LLM 独立打分，不合格自动修改，把翻译质量校验从"人工检查"变成"自动保证"
- **仓库隔离，安全提交**：目标仓库 clone 到项目外部独立缓存；只创建分支提 PR，绝不碰 master，零误操作风险
- **公式自动转 SVG**：LaTeX 公式（$$...$$ / \\(...\\) / \\[...\\]）转为 codecogs SVG 图片，博客端无需 MathJax 也能正确渲染
- **容错不断线**：任意步骤出错时提示重试或跳过，不会因单步失败导致全流程崩溃
- **随时中断，随时继续**：PR 提交前有确认步骤，也可以跳过 GitHub 环节，blog_output 文件已生成可稍后手动上传

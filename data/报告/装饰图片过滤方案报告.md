# 微信公众号装饰图片自动过滤 — 方案分析报告

> 问题：下载微信公众号文章时，模板装饰图片（顶部横幅、分割线、底部二维码等）会连同正文配图一起保存，目前只能靠人工审核删除。
> 核心洞察：同一公众号模板的装饰图片是固定且反复出现的，因此不需要每次调用大模型 API 去识别。

**日期**：2026-07-17

---

## 1. 问题定义

在 `src/downloader.py` 的 `_parse_article_content()` 方法中，程序无差别下载正文区域内的所有 `<img>` 标签：

```python
# downloader.py:171
all_imgs = content_div.find_all('img')
```

这导致装饰性图片（顶部/底部横幅、分割线、引导关注二维码等）与真正的正文配图（图表、截图、公式图）混在一起，最终进入翻译和博客转换流程。

### 关键事实

1. **同一公众号模板的装饰图是固定的**——同一批图片在不同文章中反复出现
2. **装饰图的图片内容完全一致**（像素级相同），只是每次的 CDN URL 可能不同
3. **正文配图则每篇不同**——图表、截图、照片等各不相同

这意味着：**本质上不是图像识别问题，而是已知图片去重/过滤问题。**

---

## 2. 方案对比

### 2.1 方案一：感知哈希（pHash）— 推荐首选 ⭐

#### 原理

对每张下载的图片计算感知哈希（perceptual hash），与预先建立的"装饰图黑名单哈希库"逐条比对。相同图片产生相同哈希，直接过滤。

#### 为什么适用

- 微信模板装饰图是同一批静态资源，图片像素内容完全一致
- pHash 对完全相同或几乎相同的图片能做到 100% 准确匹配
- 即使图片被轻微压缩/缩放（汉明距离 ≤ 5），pHash 仍能可靠匹配
- 无需训练，无需 GPU，纯 CPU 运算

#### 实现

**依赖**：`pip install imagehash Pillow`

**第一步：建立哈希库（一次性，~30 分钟）**

```python
import imagehash
from PIL import Image
import json
import os

def build_hash_library(history_output_dir="output", output_file="models/decorative_hashes.json"):
    """
    扫描历史下载的所有文章，人工标记装饰图，建立哈希黑名单。
    这一步只需做一次。
    """
    known_hashes = {}

    # 遍历所有历史文章的 images 目录
    for article_dir in os.listdir(history_output_dir):
        img_dir = os.path.join(history_output_dir, article_dir, "images")
        if not os.path.isdir(img_dir):
            continue

        for img_file in os.listdir(img_dir):
            img_path = os.path.join(img_dir, img_file)
            img_hash = imagehash.phash(Image.open(img_path))
            hash_hex = str(img_hash)

            if hash_hex not in known_hashes:
                # 展示图片，人工判断是否为装饰图
                # （实际操作中，你会打开图片看一眼然后标记）
                print(f"新图片: {img_path}, 哈希: {hash_hex}")
                label = input("装饰图? (y/n): ").strip().lower()
                if label == 'y':
                    known_hashes[hash_hex] = {
                        "hash": hash_hex,
                        "label": input("标签（如 banner_top, divider, qr_code）: "),
                        "first_seen": img_path,
                    }

    with open(output_file, 'w', encoding='utf-8') as f:
        json.dump(list(known_hashes.values()), f, ensure_ascii=False, indent=2)

    print(f"已建立哈希库: {len(known_hashes)} 张装饰图 → {output_file}")
```

**第二步：过滤逻辑（集成到 downloader.py）**

```python
import imagehash
from PIL import Image

class DecorativeFilter:
    """基于感知哈希的装饰图过滤器"""

    def __init__(self, hash_library_path="models/decorative_hashes.json", threshold=5):
        with open(hash_library_path, 'r', encoding='utf-8') as f:
            self.known_hashes = json.load(f)
        self.threshold = threshold  # 汉明距离阈值，≤5 视为匹配

    def is_decorative(self, image_path: str) -> tuple:
        """
        判断图片是否为装饰图。
        返回: (is_decorative: bool, matched_label: str | None)
        """
        img_hash = imagehash.phash(Image.open(image_path))

        for entry in self.known_hashes:
            distance = img_hash - imagehash.hex_to_hash(entry["hash"])
            if distance <= self.threshold:
                return True, entry.get("label", "unknown")

        return False, None
```

#### 性能

| 指标 | 数值 |
|------|------|
| 单张图片哈希计算 | < 10ms（纯 CPU） |
| 比对速度（100 条库） | < 1ms |
| 内存占用 | 可忽略 |
| 准确率（对相同图） | 100% |
| 准确率（对变体图） | 取决于变体程度 |

#### 优点

- 实现简单，~50 行代码
- 零训练成本
- 对完全相同图片 100% 准确
- 可随时手动扩充哈希库

#### 局限

- 只能识别与库中哈希完全一致（或几乎一致）的图片
- 如果换了新模板、新装饰图，需手动添加到库
- 对大幅度修改的变体图（换色、加文字等）无效

### 2.2 方案二：预训练 CNN 特征 + 相似度匹配

#### 原理

用一个预训练卷积神经网络（如 ResNet50、EfficientNet-B0）作为特征提取器，将图片映射为高维特征向量。比对两张图片的特征向量余弦相似度，判断是否属于同类。

#### 适用场景

当装饰图不是像素级相同，而是"同一类设计元素"时——比如同一风格的分割线有多个颜色变体，pHash 无法覆盖。

#### 实现

```python
import torch
import torchvision.models as models
import torchvision.transforms as transforms
from PIL import Image
import numpy as np

class CNNFeatureFilter:
    """基于预训练 CNN 特征的装饰图过滤器"""

    def __init__(self, known_images_dir="models/decorative_templates/"):
        # 加载预训练模型（只用特征提取部分）
        self.model = models.efficientnet_b0(weights="EfficientNet_B0_Weights.DEFAULT")
        self.model.classifier = torch.nn.Identity()  # 去掉分类头
        self.model.eval()

        self.transform = transforms.Compose([
            transforms.Resize((224, 224)),
            transforms.ToTensor(),
            transforms.Normalize(mean=[0.485, 0.456, 0.406],
                                 std=[0.229, 0.224, 0.225]),
        ])

        # 预计算所有已知装饰图的特征向量
        self.known_features = self._build_feature_library(known_images_dir)

    def _extract_features(self, image_path: str) -> np.ndarray:
        img = Image.open(image_path).convert("RGB")
        tensor = self.transform(img).unsqueeze(0)
        with torch.no_grad():
            features = self.model(tensor)
        return features.squeeze().numpy()

    def _build_feature_library(self, base_dir: str) -> list:
        library = []
        for fname in os.listdir(base_dir):
            if fname.lower().endswith(('.png', '.jpg', '.jpeg', '.webp')):
                fpath = os.path.join(base_dir, fname)
                library.append({
                    "name": fname,
                    "features": self._extract_features(fpath),
                })
        return library

    def is_decorative(self, image_path: str, threshold: float = 0.90) -> tuple:
        features = self._extract_features(image_path)
        for entry in self.known_features:
            similarity = np.dot(features, entry["features"]) / (
                np.linalg.norm(features) * np.linalg.norm(entry["features"])
            )
            if similarity >= threshold:
                return True, entry["name"]
        return False, None
```

#### 性能

| 指标 | 数值 |
|------|------|
| 单张图片特征提取 | ~20-50ms（CPU）/ ~2-5ms（GPU） |
| 模型文件大小 | ~20MB（EfficientNet-B0） |
| 准确率 | 对变体有一定鲁棒性，但需要调阈值 |

#### 优点

- 对缩放、裁剪、轻微调色有鲁棒性
- 不需要训练，只需前向推理
- 比 pHash 覆盖范围更广

#### 局限

- 比 pHash 慢
- 需要 PyTorch/TensorFlow 依赖
- 阈值的设定需要反复调试

### 2.3 方案三：微调 MobileNetV3 二分类模型（自己训练）

#### 原理

用标注过的装饰图/内容图数据集，在预训练模型基础上微调（transfer learning），得到一个二分类器：输入图片 → 输出"装饰图"概率。

#### 为什么选 MobileNetV3-Small

| 模型 | 参数量 | 推理速度(CPU) | 适合场景 |
|------|--------|--------------|---------|
| MobileNetV3-Small | 2.5M | ~10ms/张 | 移动端/嵌入式/本地 |
| EfficientNet-B0 | 5.3M | ~20ms/张 | 服务器/精度优先 |
| ResNet50 | 25.6M | ~50ms/张 | 过重，不推荐 |
| ViT-B/16 | 86M | ~200ms/张 | 过重，需要大数据 |

**推荐 MobileNetV3-Small**：轻量、快速、精度对本任务足够。

**3. 推理（集成到 downloader.py）**

```python
import onnxruntime as ort
import numpy as np
from PIL import Image

class ModelBasedFilter:
    """基于微调模型的装饰图过滤器"""

    def __init__(self, model_path="models/decorative_classifier.onnx",
                 confidence_threshold=0.5):
        self.session = ort.InferenceSession(model_path)
        self.threshold = confidence_threshold

    def is_decorative(self, image_path: str) -> tuple:
        # 预处理
        img = Image.open(image_path).convert("RGB").resize((224, 224))
        img_array = np.array(img, dtype=np.float32) / 255.0
        img_array = (img_array - np.array([0.485, 0.456, 0.406])) / \
                     np.array([0.229, 0.224, 0.225])
        img_array = img_array.transpose(2, 0, 1)[np.newaxis, ...]

        # 推理
        output = self.session.run(None, {"input": img_array})
        prob_decorative = float(output[0][0][1])  # 是装饰图的概率

        return prob_decorative >= self.threshold, prob_decorative
```

## 3. 推荐策略：分层过滤

在实际项目中，建议采用**"pHash 精确匹配 + 模型概率判断"**的两层过滤架构：

```
┌─────────────────────────────────────────────────────┐
│                  图片下载完成                         │
└─────────────────────┬───────────────────────────────┘
                      │
                      ▼
         ┌────────────────────────┐
         │  第 1 层: pHash 比对    │  ← 毫秒级，零成本
         │  与已知装饰图哈希库比对  │
         └────────────┬───────────┘
                      │
            ┌─────────┴─────────┐
            │                   │
        匹配成功              匹配失败
            │                   │
            ▼                   ▼
      直接过滤         ┌────────────────────┐
                      │  第 2 层: 模型推理   │  ← 仅对未知图片启用
                      │  二分类模型判断       │
                      └────────┬───────────┘
                               │
                     ┌─────────┴─────────┐
                     │                   │
                概率 ≥ 0.8           概率 < 0.8
                     │                   │
                     ▼                   ▼
                 自动过滤          保留 + 记录日志
                                 （低置信度交给人工）
```

### 为什么这样设计

1. **pHash 挡掉 80-90% 流量**——大多数装饰图是反复出现的同一批图片，哈希匹配瞬间命中
2. **模型处理剩余 10-20%**——新模板、新变体、或者恰好与正文图片哈希冲突的情况
3. **低置信度交给人工**——当模型不确定时（0.4-0.6），保留图片让用户判断，避免误删正文配图
4. **持续学习**：人工判断的结果可以加入训练集，定期重新微调，模型越来越准

### 完整集成示例

```python
# src/decorative_filter.py — 新文件

import imagehash
import onnxruntime as ort
from PIL import Image
import numpy as np
import json
import os
import logging

logger = logging.getLogger(__name__)


class HybridDecorativeFilter:
    """pHash + 微调模型 混合装饰图过滤器"""

    def __init__(self,
                 hash_library="models/decorative_hashes.json",
                 model_path="models/decorative_classifier.onnx",
                 hash_threshold=5,
                 model_threshold=0.8,
                 uncertain_threshold=0.4):
        """
        参数:
            hash_library:       pHash 黑名单库路径
            model_path:         ONNX 模型路径
            hash_threshold:     哈希汉明距离阈值（≤此值视为匹配）
            model_threshold:    模型高置信度阈值（≥此值自动过滤）
            uncertain_threshold: 模型低置信度阈值（≤此值保留，之间的记录日志）
        """
        # pHash 层
        self.hash_threshold = hash_threshold
        self.known_hashes = []
        if os.path.exists(hash_library):
            with open(hash_library, 'r', encoding='utf-8') as f:
                self.known_hashes = json.load(f)

        # 模型层
        self.model_threshold = model_threshold
        self.uncertain_threshold = uncertain_threshold
        self.session = None
        if os.path.exists(model_path):
            self.session = ort.InferenceSession(model_path)

    def should_filter(self, image_path: str) -> tuple:
        """
        判断图片是否应该被过滤。
        返回: (should_filter: bool, reason: str, confidence: float)
        """
        # 第 1 层：pHash
        img_hash = imagehash.phash(Image.open(image_path))
        for entry in self.known_hashes:
            distance = img_hash - imagehash.hex_to_hash(entry["hash"])
            if distance <= self.hash_threshold:
                return True, f"pHash 匹配: {entry.get('label', 'unknown')}", 1.0

        # 第 2 层：模型推理（如果可用）
        if self.session is not None:
            prob = self._model_predict(image_path)
            if prob >= self.model_threshold:
                return True, f"模型判定 (置信度 {prob:.2f})", prob
            elif prob <= self.uncertain_threshold:
                return False, f"非装饰图 (置信度 {1-prob:.2f})", prob
            else:
                # 不确定区域 — 保留图片，记录日志
                logger.info(f"低置信度，保留人工判断: {image_path} (装饰图概率: {prob:.2f})")
                return False, f"不确定，保留人工判断 (概率 {prob:.2f})", prob

        return False, "未匹配任何装饰图规则", 0.0

    def _model_predict(self, image_path: str) -> float:
        """模型推理，返回装饰图概率"""
        img = Image.open(image_path).convert("RGB").resize((224, 224))
        img_array = np.array(img, dtype=np.float32) / 255.0
        img_array = (img_array - np.array([0.485, 0.456, 0.406])) / \
                     np.array([0.229, 0.224, 0.225])
        img_array = img_array.transpose(2, 0, 1)[np.newaxis, ...]
        output = self.session.run(None, {"input": img_array})
        return float(output[0][0][1])
```

## 5. 与调用大模型 API 的对比

| 维度 | 大模型 API（GPT-4o 等） | pHash | 微调 MobileNetV3 |
|------|------------------------|-------|-----------------|
| 每张图片成本 | ¥0.01-0.05 | ¥0 | ¥0 |

| 需要网络 | 是 | 否 | 否 |

**结论**：当识别目标是固定且反复出现的内容时，本地化方案（pHash + 小模型）在成本、速度、稳定性上全面优于调用大模型 API。

## 6. 小结

1. **你不是真正需要"图像识别"**——装饰图是固定的、重复的，这是已知图片过滤问题
2. **pHash 是 80% 的解决方案**——实现简单、零成本、100% 准确、30 分钟搞定
3. **微调模型是剩余 20% 的兜底**——个人完全有条件训练，Google Colab 免费 GPU 即可，15 分钟训练完成
4. **分层架构是最优解**——pHash 挡掉大部分，模型处理边界案例，低置信度交人工
5. **训练成果是一个 .onnx 文件**（10-20MB），可放入 Git 仓库，与 Python 项目无缝集成
6. **彻底消除 API 成本**——每次过滤完全本地执行，毫秒级响应，零网络依赖
